Hugo

This task initially appeared challenging and overwhelming, but by the end, it seemed straightforward and simple.

I began by installing Hugo and creating two GitHub repositories named “hugo” and “adarshs05.github.io.” I cloned both repositories into my Linux terminal. Next, I navigated to the “hugo” repository and used the command “hugo new site sreeniche” to create a new directory containing all the necessary tools and files to build the website.

To gain a better understanding of the process, I referred to a YouTube tutorial. I imported several themes from the Hugo website into my “themes” directory and selected the “Beautiful Hugo” theme as my default theme in the “hugo.toml” file. I also made some modifications in the hugo.toml file, such as updating the “baseURL” to “adarshs05.github.io” and changing the title of the homepage. Running “hugo server” in the Linux terminal was successful, and I proceeded to create a post using the “hugo new posts/your-post-title” command. Up to this point, I didn’t encounter any issues.

The most challenging part was hosting the website on GitHub. I added the remote origin as “adarshs05.github.io” using the “git remote add origin” command. There were issues related to switching between branches, specifically “master” and “main.” I resolved this issue by merging the branches.

I navigated to the “public” directory and ran the command “hugo -t beautifulhugo” to generate the necessary files for GitHub. Using the sequence of commands “git add .,” “git commit,” and “git push origin main,” I pushed the files. Initially, the changes did not reflect on GitHub, and the website loaded without a theme, resulting in errors. Finally, after a few more attempts, all issues were resolved, and I successfully was able to complete the task.
